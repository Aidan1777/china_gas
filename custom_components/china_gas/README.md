# 中国燃气 (China Gas)

[![hacs_badge](https://img.shields.io/badge/HACS-Custom-41BDF5.svg)](https://github.com/hacs/integration)

Home Assistant 自定义集成，查询中国燃气账户余额、用气记录和阶梯计费信息。

## 功能

- 🔥 查询燃气账户余额（m³）  
- 📊 月度用气量 & 费用统计  
- 🪜 阶梯计费支持（可配置三档气量 & 价格）  
- 📅 年月日级别的用气趋势数据  
- ⏱️ 每 2 小时自动刷新  
- 🧩 通过 UI 配置（config_flow），无需编辑 YAML  

## 安装

### 方式一：HACS

1. 在 HACS → 集成 → 右上角菜单 → 自定义存储库  
2. 添加 `https://github.com/Aidan1777/china_gas`，类型选「集成」  
3. 搜索「中国燃气」并安装  
4. 重启 Home Assistant  

### 方式二：手动安装

```bash
cd /path/to/homeassistant/config/custom_components
git clone https://github.com/Aidan1777/china_gas.git china_gas
```

重启 Home Assistant 后，在「设置 → 设备与服务 → 添加集成」搜索「中国燃气」。

## 配置

### 步骤一：账户信息

| 参数 | 说明 |
|------|------|
| `user_id` | 中国燃气用户 ID |
| `access_token` | 访问令牌 |
| `x_mas_app_info` | 小程序 AppInfo（抓包获取） |
| `cust_code` | 燃气客户编号 |
| `cust_name` | 燃气客户姓名 |

### 步骤二：计费配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 是否预付费 | 否 | 预付费/后付费模式 |
| 阶梯起始日期 | `0101` | 格式 MMDD，如 `0401` 表示 4 月 1 日起算 |
| 一档气量上限 | `480` m³ | 超过进入第二档 |
| 二档气量上限 | `660` m³ | 超过进入第三档 |
| 一档气价 | `2.18` 元/m³ | |
| 二档气价 | `2.62` 元/m³ | |
| 三档气价 | `3.27` 元/m³ | |

## Sensor 属性

实体主值：`balance`（燃气余额，单位 m³）

### 账户信息

| 属性 | 说明 |
|------|------|
| `address` | 地址 |
| `card_no` | 卡号 |
| `meter_no` | 表号 |
| `owe_money` | 欠费金额（元） |

### 计费信息

| 属性 | 说明 |
|------|------|
| `当前档位` | 当前所处阶梯档位（1/2/3） |
| `当前气价` | 当前阶梯气价（元/m³） |
| `年阶梯累计用气量` | 本阶梯周期累计用量（m³） |
| `阶梯剩余量` | 当前档位剩余额度（m³） |

### 统计信息

| 属性 | 说明 |
|------|------|
| `本年度日均` | 本年度日均用气量（m³） |
| `近3月日均` | 近 3 个月日均用气量（m³） |
| `近12月日均` | 近 12 个月日均用气量（m³） |
| `年预估用气量` | 基于日均推算的全年用气量（m³） |
| `年预估费用` | 基于阶梯计费的年预估费用（元） |
| `预计可用天数` | 当前余额预计可用天数 |

### 月度记录

| 属性 | 说明 |
|------|------|
| `月用气记录` | 每月用气量 JSON（`{"2025-06": 25.3, ...}`） |
| `月费用记录` | 每月费用 JSON |
| `f_gas_total` | 每月燃气总量（用于图表） |
| `e_gas_total` | 每月估算用量（用于图表） |

### 图表数据

| 属性 | 说明 |
|------|------|
| `monthlist` | 月度列表，用于仪表盘月视图 |
| `yearlist` | 年度列表，用于仪表盘年视图 |
| `daylist` | 每日列表，用于弹窗日视图 |

### 最近记录

| 属性 | 说明 |
|------|------|
| `last_payment` | 最近一次缴费金额 |
| `last_payment_time` | 最近一次缴费时间 |
| `last_record` | 最近一次购气量（m³） |
| `last_record_time` | 最近一次购气时间 |

## 技术实现

- 模拟微信小程序环境发起请求  
- 每次刷新调用 3 个接口，间隔 5 秒避免频率限制  
- 数据处理：按月度汇总购气记录 → 日均推算 → 阶梯费用估算
